Gotchas & best practices
- If pin_memory=True in your DataLoader, non_blocking=True is good; if not, set non_blocking=False.
- Ensure tensors added to TensorBatchStorage have matching shapes on all dims except cat_dim.
- The final partial window (< K batches) IS flushed at the end of each epoch,
  so short epochs and trailing batches are fully accounted for in losses,
  outputs and targets. (Older versions silently dropped them.)
- keep_outputs=False saves RAM but makes compute_metrics receive (None, None).
- scheduler is called (stepped) after every epoch.
- history[i]['train_metrics'] / ['test_metrics'] are zlib-compressed pickles
  (SaveDump blobs); decode them with MLTools.Utilities.LoadDump.
- The best checkpoint (best_path) is written on the FIRST measured epoch as
  the baseline, then re-written whenever quantify() improves by at least
  min_rel_improval (relative) over the best value so far.
- Resuming with MORE epochs than the original run: load_checkpoint restores
  the scheduler's state_dict verbatim, which includes its hyper-parameters
  (e.g. CosineAnnealingLR's T_max). A cosine schedule that already annealed
  to ~0 stays at ~0 for the extra epochs. If you extend a run, rebuild the
  scheduler for the new horizon and override load_checkpoint to skip (or
  remap) the saved scheduler state.

# ============================================================
# Helper classes (same module: MLTools.Utilities.TrainingUtils)
# ============================================================
#
# RecentLossCounter(memory: int = 10)
#   Sliding-window mean over the last `memory` scalar values.
#   Methods: add(val: float), value() -> float (0.0 when empty), __len__.
#
# AverageLossCounter()
#   Plain running mean over everything added so far.
#   Methods: add(val: float), value() -> float (0.0 when empty), __len__.
#
# TensorBatchStorage(cat_dim: int = 0, auto_unsqueeze_scalars: bool = True)
#   Accumulates detached tensors (on their original device, typically GPU) and
#   concatenates them into ONE CPU tensor on value(). Scalars (0-D) are
#   auto-unsqueezed to 1-D so losses can be stacked.
#   Methods: add(t), value() -> Tensor (empty 1-D float32 when empty),
#            clear(), __len__.
#
# DelayedStorage(itemize_every_k: int = 10, cat_dim: int = 0,
#                auto_unsqueeze_scalars: bool = True)
#   Buffers K tensors, then emits ONE concatenated CPU tensor (1 device->host
#   transfer per K batches). add() returns None until the K-th call, which
#   returns the concatenated CPU tensor.
#   Methods:
#     add(t) -> Optional[Tensor]
#     flush() -> Optional[Tensor]
#         Emits whatever is buffered (fewer than K items) and clears the
#         buffer; returns None when nothing is buffered. TrainingLoop calls
#         this at the end of each epoch.
#     last_value() -> Optional[Tensor]
#
# CosineAnnealingWithWarmup(optimizer, warmup_epochs, total_epochs,
#                           warmup_factor=0.01)
#   Convenience builder: LinearLR warmup (warmup_factor -> 1.0 over
#   warmup_epochs) chained via SequentialLR into CosineAnnealingLR
#   (T_max = total_epochs - warmup_epochs).
#   Validation: warmup_epochs >= 1 and total_epochs > warmup_epochs,
#   otherwise ValueError.

# ============================================================
# Class: TrainingLoop
# ============================================================
# A flexible and extensible training pipeline wrapper for PyTorch.
# Supports checkpointing, metric computation, loss tracking,
# and batched GPU→CPU logging.
#
# Args:
#   model (nn.Module): Model to be trained.
#   optimizer (torch.optim.Optimizer): Optimizer for parameter updates.
#   criterion (Callable): Loss function.
#   device (torch.device): Target device (e.g., 'cuda' or 'cpu').
#   epochs (int): Number of epochs to train.
#   train_loop_constructor (Callable): Function returning a tqdm iterator for training data.
#   test_loop_constructor (Callable): Function returning a tqdm iterator for test data.
#   log_every_k (int): Batching frequency for CPU transfers (reduces I/O cost).
#   recent_memory (int): Number of recent batches used for running average.
#   scheduler (torch.optim.lr_scheduler._LRScheduler, optional): Learning rate scheduler.
#       Defaults to a constant LambdaLR when None.
#   checkpoint_path (str, optional): Path to save checkpoints each epoch.
#   best_path (str, optional): Path to save the best-performing model.
#   non_blocking (bool): Enables non-blocking CUDA transfers. Default True.
#   keep_outputs (bool): Whether to retain all outputs/targets for metric computation.
#   metrics_train (bool): Compute metrics over training outputs. Default False.
#   metrics_test (bool): Compute metrics over test outputs. Default True.
#   min_rel_improval (float): Minimal relative improvement of quantify() that
#       counts as a new best (floored by eps=1e-8 in absolute terms). Default 0.005.
#
# State:
#   epoch (int): Index of the epoch currently being executed (0-based).
#   best_val (float): Best quantify() result so far; math.inf before the
#       first measured epoch.
#   history (List[dict]): One entry per epoch with keys:
#       'epoch' (int), 'train_loss' (float), 'test_loss' (float),
#       'train_metrics' (SaveDump blob), 'test_metrics' (SaveDump blob).
#   loaded (bool): True after a successful load_checkpoint().
#
# Hooks to Override:
#   train_batch(data) -> (loss, outputs, targets):
#       Defines a single training step. Can be customized for complex batches.
#
#   test_batch(data) -> (loss, outputs, targets):
#       Defines a single evaluation step. The default implementation runs
#       under torch.inference_mode(); if you override it, wrap your code in
#       torch.inference_mode() or torch.no_grad() yourself.
#
#   compute_metrics(outputs, targets, valid_mode) -> Any:
#       Computes metrics at epoch end. Receives concatenated CPU tensors
#       (or None when valid_mode is False). Must return ONE pickle-able value.
#
#   improved_val(old_val, new_val) -> None:
#       Called when the best value improves. Default prints a message.
#
#   pre_epoch() -> None:
#       Called before each epoch begins.
#
#   post_epoch(history: List[dict]) -> Optional[bool]:
#       Called at the end of every epoch. Can implement early stopping.
#       Return a truthy value to stop training after the current epoch.
#       Exceptions raised here are caught and printed, not propagated.
#
#   print_epoch_results(epoch, train_loss, train_metrics, test_loss, test_metrics) -> None:
#       Defines how results are displayed after each epoch.
#
#   update_pbar(loop, mode, epoch, avg_loss, recent_loss) -> None:
#       Updates the progress bar display for train/test phases.
#
#   quantify(test_loss, test_metrics) -> float:
#       Produces a single scalar score used to track the "best" model.
#       Lower values are considered better.
#       By default: return test_loss
#
# Internal Methods:
#   load_checkpoint(path): Loads model, optimizer, scheduler states, and history.
#       Uses torch.load(..., weights_only=False): only load checkpoints you trust.
#   save_checkpoint(path): Saves model, optimizer, scheduler states, and history.
#   _train_step(epoch): Executes one training epoch (flushes partial buffers).
#   _test_step(epoch): Executes one evaluation epoch (flushes partial buffers).
#   _improvement_check(x, prev_x, minimal_change, eps=1e-8): static helper,
#       True when x <= prev_x - max(|prev_x| * minimal_change, eps).
#   run(resume=True): Main training entry point. With resume=True and a
#       checkpoint_path, continues from the stored epoch+1.
#
